## Highlights

- Hardened the CLI and config-loading path so invalid input, unreadable `--command-config` files, and incomplete SASL shortcut configuration fail fast instead of appearing successful.
- Fixed several plan/apply correctness issues, including topic replication decreases, Kafka Streams internal topic handling with custom `application-id`, and duplicate ACL generation.
- Tightened validation for desired state files by rejecting invalid topic numeric values and unresolved custom ACL owners before plan or apply work starts.
- Refreshed documentation and Confluent Cloud examples so the published CLI surface, environment-variable examples, and sample commands match the current tool behavior.
- Improved maintenance ergonomics with unique temp files in integration tests, aligned Jackson/logging dependencies, weekly Dependabot review, and a slimmer non-root Docker runtime image.

## Upgrade Notes

- Invalid CLI invocations now return a non-zero exit code.
- An explicit `--command-config` path must exist and be readable.
- Topic definitions now require `partitions >= 1`, and explicit `replication` values must be `>= 1`.
- `customServiceAcls` and `customUserAcls` entries must reference defined services and users.
- The Docker image now runs as an unprivileged `kafka-gitops` user and uses `kafka-gitops` as the container entrypoint.
